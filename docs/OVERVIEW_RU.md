## Обзор

### Пайплайн предварительной обработки

*GaMAC* представляет свой собственный предобработчик данных, который преобразует
мультимодальные данные в единый вектор данных, используемый для дальнейшей кластеризации. 

Вот схема верхнего уровня:

По сути, конвейерный процессор работает с таблицами, изображениями и текстами. 
Для предварительной обработки текста предусмотрены следующие этапы
* **Категориальные объекты** преобразует категориальные объекты с помощью кодировки OneHot или Label
* **Столбцы NaN** обрабатывают NaN, заполняя собственные средние арифметические значения или значения моды, или отбрасывая объекты с определенного порога доли NaN
* **Масштабирование** пересчитывает числовые столбцы с новым диапазоном чисел
* **Финализация** с преобразованием всех данных в массив numpy

Для изображений и текстов:
* **Предварительная обработка на основе клипов** преобразует изображения и тексты в эмбеддинги модели [openai/clip-vit-large-patch14](https://huggingface.co/sergeyzh/rubert-tiny-turbo), которые объединяются с табличными данными

Будущая работа: разработка возможности кастомизации эмбеддера.

### Пайплайн выбора меры

На данном этапе для задачи кластеризации используется мета-классификатор показателей меры. 
Этот классификатор позволяет выбрать оптимальную меры для конкретного набора данных. 
На выходе конвейера выдается конкретная мера, которая будет использоваться при поиске наилучшего алгоритма кластеризации.

### Пайплайн оптимизации

В этом пайплайне используются 6 алгоритмов кластеризации, адаптированных для ускорения на GPU. 
Эти алгоритмы перебираются с различными вариантами гиперпараметров, которые использует Optuna framework.
Результаты пайплайна: наилучшая конфигурация алгоритма кластеризации с оптимальными гиперпараметрами, наилучшее значение показателя меры качества и история поиска конфигурации оптимального алгоритма.
